Modules

Anonymous function declaration / invocation

(function () {
    var foo = 'bar';
}());
	
myModule = (function () {
    var foo = 'bar';
    function doSomething (value) {
        foo = value;
    }

    return {
        doSomething: doSomething
    };
}());

myModule.doSomething();
Exposes some functions 
MyClass = (function () {
    function MyConstructor (foo, bar) {
        this.myMethod = ...
    }

    return MyConstructor;
}());

new MyClass(foo, bar);
Defines a constructor 
MyClass = (function ($, myDependency) { // <-- parameters
    function MyConstructor (foo, bar) {
        this.myMethod = ...
    }

    return MyConstructor;
}(jQuery, MyOtherModule)); // <-- arguments

new MyClass(foo, bar);
Importing via function arguments

Good modules:

  • Are lightweight and specific
  • Are particular about what they return / expose
  • Avoid global interaction

Libraries

  • MVC-style structure: models, views, routing
  • Model & collection events

  • Module loader
  • Separates modules into their own files
  • Module "importing" and dependency management
